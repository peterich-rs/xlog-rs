# Rust Preview 0.1.0-preview.2

This preview is focused on `mars-xlog-core`, the low-level runtime crate behind
`mars-xlog`.

Highlights:

- optimize low-level core paths around append scheduling, buffer/file handling,
  and recovery flow
- harden single-writer and sync-recovery behavior in the file manager
- expand `mars-xlog-core` regression coverage across async engine, buffer,
  compression, crypto, mmap, and one-shot flush paths
- keep the published Rust MSRV at Rust 1.85

Scope notes:

- `mars-xlog-core` is still the implementation-layer crate; `mars-xlog` remains
  the default Rust entry point for most users
- this is still a Preview release, not a GA stability claim
